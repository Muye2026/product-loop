# 说明

`product-loop` 是一个面向小型硬件产品的开源 skill，用来把“产品想法 -> 方向探索 -> 设计收敛 -> 结构化设计包 -> 可选 CAD 迭代”串成一条稳定流程。

它适合：

- 小型消费电子
- 桌面硬件
- 轻量机器人外设
- 需要规范交接物的概念到规格流程
- 工业设计前需要先明确元器件、模块和包装约束的产品流程

它不适合：

- 量产 DFM 深化
- 模具和工装设计
- 公差链工程
- 大型机械系统
- 只靠一张渲染图直接生成最终 CAD

## 核心原则

这个 skill 固定遵守三条规则：

1. 先做环境检测。
2. 在造型前调研同类产品和可落地模块。
3. 用 `Design Pack` 作为概念阶段和下游建模之间的统一接口。
4. 仓库边界保持干净：可安装 skill、references、scripts、examples、仓库文档彼此分开。

## Skill 在做什么

主流程分成 6 个主阶段，并包含 1 个可选架构门和 1 个必做的元器件/模块预选型阶段：

1. `Phase 0: Environment Check`
2. `Phase 1: Brief Clarification`
3. `Phase 1.5: Module Architecture Framing`，当模块选择会明显影响产品形态时触发
4. `Phase 1.6: Research-backed Component Selection`
5. `Phase 2: Direction Exploration`
6. `Phase 3: Visual Convergence`
7. `Phase 4: Design Translation`
8. `Phase 5: CAD Loop or Handoff`

环境检测会把执行路径分成三种模式：

- `full`：有 CAD 能力，继续进入参数化草模循环
- `spec-only`：没有 CAD，只输出设计包和评审物
- `handoff`：环境或输入不完整，只输出交接包

## 标准输出

完整的 `spec-only` 和 `full` 路径应输出：

- `Reference Cases`
- `Candidate Components`
- `Selected or Assumed Components`
- `Packaging Constraints`
- `Concept Directions`
- `Render Set`
- `Appearance Spec`
- `Structure Spec`
- `Design Pack`
- `Review Report`

当输入或环境能力不完整时，`handoff` 模式可以更早停止，此时输出：

- `Normalized Brief`
- `Assumptions`
- `Open Questions`
- `Handoff Brief`

`full` 模式再追加：

- `CAD Iteration Inputs`
- `Parametric Draft Model`
- `CAD Iteration Report`

## Design Pack 接口

第一版 `Design Pack` 固定字段如下：

- `product_goal`
- `execution_mode`
- `hard_constraints`
- `component_envelopes`
- `reference_cases`
- `component_requirements`
- `component_candidates`
- `selected_components`
- `packaging_constraints`
- `sourcing_risks`
- `layout_zones`
- `mounting_strategy`
- `style_features`
- `manufacturing_risks`
- `forbidden_features`
- `acceptance_checks`

skill 自带 3 个 Python helper scripts，用来做设计包归一化、评审矩阵生成和交接文档生成。

## 安装

把内部的 `product-loop/` 目录复制或软链接到 Codex 的 skills 目录。

示例：

```bash
cp -R product-loop/product-loop "${CODEX_HOME:-$HOME/.codex}/skills/"
```

安装后，Codex 应能以 `$product-loop` 发现这个 skill。

## Helper scripts

包含 3 个轻量 Python 工具：

- [`normalize_design_pack.py`](./product-loop/scripts/normalize_design_pack.py)
- [`build_review_matrix.py`](./product-loop/scripts/build_review_matrix.py)
- [`emit_handoff_brief.py`](./product-loop/scripts/emit_handoff_brief.py)

用法示例：

```bash
python3 product-loop/scripts/normalize_design_pack.py \
  examples/desktop-device-demo/design-pack.json \
  /tmp/design-pack.normalized.json

python3 product-loop/scripts/build_review_matrix.py \
  examples/desktop-device-demo/design-pack.json \
  /tmp/review-matrix.md

python3 product-loop/scripts/emit_handoff_brief.py \
  examples/desktop-device-demo/design-pack.json \
  /tmp/handoff-brief.md
```

## 维护约束

- `SKILL.md` 保持短，把细节放到 `references/`
- 不要把本机路径、私有仓库、个人 CAD 环境写死进 skill
- 示例只作为公开演示，不作为某个具体产品的唯一真相
