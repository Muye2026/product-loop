# Appearance Spec Template

Write this spec after a direction has been selected and render work has converged.

## Required Sections

1. `Intent`
   - one-paragraph summary of the chosen direction

2. `Primary silhouette`
   - overall body type
   - aspect ratio
   - visual mass distribution
   - symmetry or asymmetry rule

3. `Front-face hierarchy`
   - focal element
   - secondary information areas
   - service or control areas
   - spacing and alignment logic

4. `Side and top read`
   - thickness behavior
   - taper, chamfer, or facet language
   - any controlled overhangs or support forms

5. `Feature language`
   - display window behavior
   - lens or sensor treatment
   - port and button expression
   - seams and split-line intent

6. `CMF guidance`
   - primary materials
   - finish intent
   - contrast strategy
   - any accent color constraints

7. `Forbidden moves`
   - visual decisions that break the selected direction

## Writing Rule

Translate from the image into reusable constraints. Do not write only mood or taste language. Make sure a downstream modeler can infer geometry priorities from this spec.
